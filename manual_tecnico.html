2. Arquitectura de la aplicación

SIDACS sigue una arquitectura modular por capas. El hilo principal de Qt ejecuta la interfaz; el procesamiento de cada frame ocurre en el mismo hilo mediante señales pyqtSignal emitidas desde temporizadores (QTimer).

main.py └── QApplication + dark_theme.qss └── ui/main_window.py (MainWindow — orquestador central) ├── ui/video_panel.py → Muestra el frame en QLabel ├── ui/control_bar.py → Botones de acción ├── ui/timeline_bar.py → Slider + marcas de eventos ├── ui/sidebar_panel.py → Métricas, alertas, log ├── video/player.py → Reproducción de archivo ├── video/webcam.py → Captura en vivo ├── video/optical_flow.py → Análisis Farneback ├── video/frame_utils.py → BGR → QPixmap └── ml/* → Dataset, ML, evidencia, stats

Patrón de comunicación: los módulos de video (VideoPlayer, WebcamCapture) emiten la señal frame_ready con un numpy.ndarray BGR. MainWindow._on_frame_ready() recibe ese frame y decide si solo mostrarlo o también analizarlo, etiquetarlo, predecir y actualizar la UI.

6. Importar y reproducir video

Para poner / reproducir un video se usó:

OpenCV VideoCapture + QTimer + pyqtSignal frame_ready

Archivo: video/player.py — clase VideoPlayer

OperaciónAPI / clase usadaDescripción técnica
Cargar archivo cv2.VideoCapture(path) Abre MP4, AVI, MKV, MOV, WMV. Si isOpened() falla, emite señal error_occurred.
Obtener FPS y duración CAP_PROP_FPS, CAP_PROP_FRAME_COUNT Calcula duración = total_frames / fps. Si FPS es 0, usa DEFAULT_FPS = 30.
Reducir latencia CAP_PROP_BUFFERSIZE = 1 Evita acumular frames en buffer interno de OpenCV; respuesta más inmediata al seek.
Reproducir QTimer con intervalo 1000 / fps ms Cada tick llama _read_next_frame() → capture.read() → emite frame_ready.
Pausar / detener QTimer.stop() + capture.release() Libera el recurso de video al cambiar de fuente o cerrar la app.
Seek (saltar en el tiempo) CAP_PROP_POS_FRAMES + read() Convierte segundos a índice de frame, posiciona el cursor y lee un frame para mostrarlo de inmediato.
Fin de video Señal playback_finished Se emite cuando read() devuelve False; la ventana principal detiene el análisis.
Posición en timeline Señal position_changed(current, total) Notifica segundos actuales y duración total para sincronizar el slider.

Flujo al importar (desde MainWindow._on_import_video)

  1. QFileDialog.getOpenFileName() — diálogo nativo de Windows para elegir archivo.
  2. VideoPlayer.load(path) — abre con OpenCV.
  3. cv2.VideoCapture(path).read() — lee el primer frame para previsualización sin iniciar reproducción.
  4. TimelineBar.set_duration() — configura el slider con la duración del clip.
  5. Se guarda _last_frame para permitir etiquetado antes de iniciar monitoreo.

9. Iniciar / detener monitoreo

Archivo: ui/main_window.py

Iniciar Monitoreo activa el pipeline completo de análisis sobre la fuente actual (archivo o webcam):

  1. Si es archivo: video_player.seek_start() + video_player.play().
  2. Marca _monitoring_active = True.
  3. Reinicia: OpticalFlowAnalyzer, SegmentTracker, SessionStatistics, buffer de evidencia, alertas del sidebar.
  4. A partir de aquí, cada frame_ready pasa por flujo óptico, ML, timeline y estadísticas.

Detener Análisis pausa el reproductor, detiene webcam si está activa, pone _monitoring_active = False y restaura el badge "EN LINEA". Los frames siguen mostrándose si el video continúa en pausa manual, pero ya no se analizan.

Reproducción sin monitoreo: tras importar un video, el usuario puede mover el slider y ver frames sin análisis. Esto permite navegar y usar Etiquetar en cualquier posición.

10. Detección de movimiento — Flujo óptico Farneback

Para detectar movimiento se usó:

cv2.calcOpticalFlowFarneback (algoritmo de Gunnar Farneback, OpenCV)

Archivo: video/optical_flow.py — clase OpticalFlowAnalyzer

Pipeline por frame

  1. Redimensionar el frame a máximo 640×360 (FLOW_PROCESS_WIDTH/HEIGHT) para procesar más rápido.
  2. Convertir a escala de grises con cv2.cvtColor(..., COLOR_BGR2GRAY).
  3. Suavizar con cv2.GaussianBlur(5×5) para reducir ruido.
  4. Comparar con el frame gris anterior almacenado en _prev_gray.
  5. Calcular flujo denso con Farneback: 
    cv2.calcOpticalFlowFarneback(prev_gray, gray, None,
    pyr_scale=0.5, levels=3, winsize=15,
    iterations=3, poly_n=5, poly_sigma=1.2, flags=0)
  6. Magnitud y ángulo con cv2.cartToPolar(fx, fy).
  7. Filtrar píxeles activos donde magnitud > FLOW_MIN_MAGNITUDE (0.4).
  8. Suavizado temporal de intensidad con media exponencial (FLOW_SMOOTH_ALPHA = 0.35).
  9. Overlay visual: flechas con cv2.arrowedLine en grilla cada 16 px; HUD con cv2.putText.

Dataclass de salida: MotionFeatures

CampoCómo se calcula
intensidad_movimiento0–100, derivada de magnitud promedio × escala 12, suavizada.
magnitud_promedioMedia de magnitud en píxeles activos.
direccion_promedioMedia circular de ángulos en grados (arctan2 de senos/cosenos medios).
cantidad_movimientoPorcentaje de píxeles activos sobre el total.
nivel_riesgoBAJO / MEDIO / ALTO según umbrales de intensidad.
motion_detectedTrue si intensidad ≥ 8 y hay píxeles activos.
annotated_frameFrame BGR original con vectores y HUD dibujados.

Colores de vectores (BGR en OpenCV)

  • Magnitud > 4.0 → rojo intenso
  • Magnitud > 2.0 → naranja
  • Resto → amarillo

14. Etiquetado manual (Normal / Sospechoso)

Para etiquetar frames se usó:

Botones colapsables PyQt5 + DatasetManager.append_manual() + análisis bajo demanda con Farneback

Interfaz — ui/control_bar.py

  • Botón toggle "Etiquetar" (QPushButton con setCheckable(True)).
  • Al activarse, muestra panel con botones "Normal" y "Sospechoso".
  • Se habilita cuando hay video cargado o webcam activa con al menos un frame visible (set_labeling_available()).

Lógica — MainWindow._on_manual_label()

  1. Obtiene características con _features_for_labeling():
    • Si hay monitoreo activo: usa _last_features del último análisis.
    • Si el video está pausado o sin monitoreo: ejecuta OpticalFlowAnalyzer.process(_last_frame) en el frame visible actual.
  2. Guarda en CSV con etiqueta elegida por el usuario (corrige al etiquetado automático heurístico).
  3. Actualiza sidebar con intensidad y riesgo del frame etiquetado.

Caso de uso: importar video → navegar con el slider → abrir Etiquetar → marcar Normal o Sospechoso en frames clave → entrenar modelo con dataset corregido.

19. Capturas PNG y clips MP4 de evidencia

Archivo: ml/evidence_manager.py

Buffer circular — clase FrameRingBuffer

AspectoImplementación
Estructuracollections.deque con maxlen calculado según FPS y segundos de buffer.
Capacidad~6 segundos antes del evento (CLIP_BEFORE_SEC = 5 + margen).
Almacenamientoframe.copy() para no mutar el array original.

Captura PNG automática

  • Condición: confianza ML ≥ EVIDENCE_SCREENSHOT_CONF (85 %).
  • API: cv2.imwrite(path, frame).
  • Carpeta: evidence/.
  • Cooldown: 3 segundos entre capturas.
  • Nombre: timestamp YYYY-MM-DD_HH-MM-SS.png.

Clip MP4 automático

  • Condición: confianza ≥ EVIDENCE_CLIP_CONF (90 %).
  • Inicio: toma frames previos del ring buffer + continúa grabando CLIP_AFTER_SEC = 5 segundos.
  • API: cv2.VideoWriter con fourcc 'mp4v'.
  • Carpeta: evidence/clips/.
